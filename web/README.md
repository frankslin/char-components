# web/ — 部件檢索新實作

取代單一 `.htm` 檔案的新版前端。不用任何 bundler/建置工具——純 ES module + `fetch()`，部署到任何靜態網頁空間後，開啟 `web/index.html` 就能用。

```
web/
├── index.html   頁面骨架（含側邊部件鍵盤面板）
├── style.css    版面樣式（含深色模式、字源色彩、響應式側邊面板）
├── core.js      比對演算法，逐句對照 legacy/部件檢索.htm 的核心程式碼翻譯而成
├── data.js      用 fetch() 載入 ../data/dt.jsonl、rt.jsonl、vt.json、kt.json
├── keypad.js    解析 kt.json 成 { icon, name, rows } 分類陣列，供側邊面板使用
├── blocks.js    字源(Unicode 區塊)分類的共用定義：色彩 class、圖例文字、篩選旗標字元
└── app.js       DOM 組裝：查詢輸入、即時搜尋、側邊鍵盤、字源圖例、結果渲染、複製到剪貼簿
```

## 為什麼不能用 `file://` 雙擊開啟

跟 `legacy/部件檢索.htm` 不同，這個新版**需要透過 http(s) 伺服器開啟**，不能直接雙擊本機檔案。原因是 `data.js` 用 `fetch()` 讀取 `../data/*`，多數瀏覽器在 `file://` 底下會擋掉這類同源請求。這是刻意的取捨：新實作的目標是「整體部署後，點開能用」，不是延續舊版「可離線雙擊」的約束。本機開發時起一個靜態伺服器即可：

```sh
python3 -m http.server 8000
# 開啟 http://localhost:8000/web/index.html
```

## 功能

- **部件輸入**：文字框輸入部件序列，或 `\字` 查該字的完整拆分樹。
- **側邊部件鍵盤**：22 個分類(標點筆畫、字頭、包圍、部首類聚等，資料來自 `data/kt.json`，逐字沿用原版 `kt` 分類定義)**一次全部列出**，不需要切換分頁——沿用 legacy 版「所有分類同時可見」的設計精神，只是改成多欄 CSS Grid 排版(`grid-template-columns: repeat(auto-fill, minmax(...))`)取代原本固定欄位的表格。側邊面板本身用 `position: sticky` 貼齊視窗高度(`calc(100vh - 3rem)`)，佔滿整個右側，內部才是可捲動區域(`overflow-y: auto`)。點擊字元會插入到輸入框目前游標位置；面板可收合(窄螢幕會自動改成主面板下方的固定高度可收合區塊)。

  **踩過的坑**：一開始用 CSS multi-column(`column-width`)排版，內容溢出時瀏覽器的行為是往右邊多長一欄，而不是往下長；但外層容器只開了 `overflow-y`，導致多出來的欄位被裁掉、永遠捲不到，使用者會覺得「鍵盤沒辦法捲動」。改用 CSS Grid 讓內容單純往下折行後才解決；另外 flex 容器裡要讓 `overflow-y: auto` 生效，中間每一層都要有 `min-height: 0`，這是 flexbox 常見的坑，光加 `overflow-y` 不夠。
- **即時查詢(自動完成)**：預設開啟，輸入時debounce 120ms 後自動查詢(IME 組字中不會誤觸發)；即時查詢用較小的結果上限(100，精確命中不受限)，避免大量結果時卡頓；按「查詢」鍵或 Enter 一律跑完整上限(3000)。
- **字源圖例**：14 個分類(基本／A~J 區／相容／補充／其他)，對應 `core.js` 的 `getBlock()` 分類。每個圖例都是可點擊的按鈕，點下去會插入「只搜此區」的篩選符號(沿用原版 `Key('@'/'A'..'J'/'X'/'Y'/'Z')` 的語意)。查詢結果每個字塊是簡單的圓角方塊，底色直接用該字源分類的顏色填滿(不疊加額外裝飾)，讓不同字源區一眼就能分辨；命中類型(精確/模糊/正體)只用文字顏色/字重/邊框樣式表達，不會跟字源底色互相干擾。
- **預設排除「相容」區**：`app.js` 在使用者完全沒有自己指定篩選符號時，會自動在查詢前面補上「除了相容以外」的區塊旗標(`DEFAULT_FLAGS`)，所以預設搜尋結果不會出現「相容表意文字」這個區塊的重複碼位。想單獨查相容字，點圖例的「相容」即可(一旦輸入裡出現任何篩選符號，就完全照使用者指定的來，不再套用這個預設值)。
- **補充字(PUA)提示**：「補充」分類的字使用作者暫定的私有造字區碼位，不是正式 Unicode 編碼，hover 這類字塊只會顯示「補充（尚未正式編碼，暫用私有碼位）」，不會顯示假的 `U+` 碼位；其餘已正式編碼的字，hover 會顯示 `U+XXXX · 分類名稱`。
- **包容異體**、**同字異拆(顯示全部拆法)**、**排除未編碼補充字**：三個核取方塊，語意對應 `core.js` 的 `v`/`d`/`u` 參數。
- 點擊結果字塊會複製到剪貼簿並顯示提示。

## 已驗證的行為（見開發時的 Node + Playwright 測試）

- `getMatch()` 對「日月／明／日日月／日明」等查詢的結果，與 `legacy/部件檢索.htm` 的演算法邏輯逐句核對一致，且維持「日明 ⊆ 日日月」等原文描述的不變量。
- 「包容異體」勾選後，輸入「咅邑」能查到「部」（`⻏→邑` 的單向映射生效）；未勾選則查不到，只多出「邑」本身。
- 「同字異拆」勾選後，`\主` 能展開出「丶王」「亠土」等全部同字異拆分支；不勾選只顯示第一種拆法。
- 側邊鍵盤 22 個分類(一次全部渲染，無分頁)、圖例 14 個分類都正確渲染；點擊鍵盤字元、圖例篩選符號都會正確插入到輸入框並觸發即時查詢。
- 即時查詢會在輸入時自動觸發(debounce)，不需要手動按查詢鍵；點擊「基本」圖例後查詢「明」，結果會被正確限縮到基本字區。
- 補充字(PUA)分類的字塊 hover title 正確顯示「尚未正式編碼」提示，而非誤導性的 `U+` 碼位。
- 預設查詢「豈」會找到 U+8C48(基本字區的「豈」)等 101 字，但不含「相容表意文字」區的 U+F900(外觀相同、但是不同碼位的相容重複字)；點圖例的「相容」後單獨查「豈」則只找到那 1 個相容碼位——證實預設排除、圖例可單獨查詢兩種行為都正確。
- `\字` 解構語法、點字複製到剪貼簿都經瀏覽器（Playwright + Chromium）實測可用，主頁字型（WFG FSung）正確載入渲染，結果依字源正確上色，沒有豆腐字或 console 錯誤。

## 跟 legacy 版比，目前刻意先不做的功能

- 沒有「異體檢索」入口（`core.js` 已經有 `getVariant()`，邏輯已就緒，只是 `app.js` 還沒接 UI）。
- 沒有 cookie/`localStorage` 記住使用者的選項狀態(每次重新整理都會回到預設值)。
- 沒有原版的「跳轉外部字典」功能（`ref` 可客製化連結），目前點字一律只複製到剪貼簿。
- `data/` 幾個檔案（共約 4MB）會一次整包載入，還沒做成 `webfonts/wfg-fsung` 那種按需分片。

這些都是後續可以再加的功能，不是遺漏或 bug；`core.js` 的比對邏輯本身已經是完整、經測試驗證的核心。
