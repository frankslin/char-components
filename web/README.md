# web/ — 部件檢索新實作

取代單一 `.htm` 檔案的新版前端。不用任何 bundler/建置工具——純 ES module + `fetch()`，部署到任何靜態網頁空間後，開啟 `web/index.html` 就能用。

```
web/
├── index.html   頁面骨架
├── style.css    版面樣式（含深色模式），引用 ../webfonts/wfg-fsung/wfg-fsung.css
├── core.js      比對演算法，逐句對照 legacy/部件檢索.htm 的核心程式碼翻譯而成
├── data.js      用 fetch() 載入 ../data/dt.jsonl、rt.jsonl、vt.json
└── app.js       DOM 組裝：查詢輸入、結果渲染、複製到剪貼簿
```

## 為什麼不能用 `file://` 雙擊開啟

跟 `legacy/部件檢索.htm` 不同，這個新版**需要透過 http(s) 伺服器開啟**，不能直接雙擊本機檔案。原因是 `data.js` 用 `fetch()` 讀取 `../data/*.jsonl`，多數瀏覽器在 `file://` 底下會擋掉這類同源請求。這是刻意的取捨（見對話紀錄）：新實作的目標是「整體部署後，點開能用」，不是延續舊版「可離線雙擊」的約束。本機開發時起一個靜態伺服器即可：

```sh
python3 -m http.server 8000
# 開啟 http://localhost:8000/web/index.html
```

## 已驗證的行為（見開發時的 Node + Playwright 測試）

- `getMatch()` 對「日月／明／日日月／日明」等查詢的結果，與 `legacy/部件檢索.htm` 的演算法邏輯逐句核對一致，且維持「日明 ⊆ 日日月」等原文描述的不變量。
- 「包容異體」勾選後，輸入「咅邑」能查到「部」（`⻏→邑` 的單向映射生效）；未勾選則查不到，只多出「邑」本身。
- 「同字異拆」勾選後，`\主` 能展開出「丶王」「亠土」等全部同字異拆分支；不勾選只顯示第一種拆法。
- `\字` 解構語法、點字複製到剪貼簿都經瀏覽器（Playwright + Chromium）實測可用，主頁字型（WFG FSung）正確載入渲染，沒有豆腐字或 console 錯誤。

## 跟 legacy 版比，目前刻意先不做的功能

這是「先求能動、再逐步補齊」的新實作起點，以下原版功能還沒有搬過來：

- 沒有「部件鍵盤」（原版用來輸入不易鍵入部件的輔助按鍵區）。
- 沒有「異體檢索」入口（`core.js` 已經有 `getVariant()`，邏輯已就緒，只是 `app.js` 還沒接 UI）。
- 沒有 cookie/`localStorage` 記住使用者的選項狀態。
- 沒有原版的「跳轉外部字典」功能（`ref` 可客製化連結），目前點字一律只複製到剪貼簿。
- 結果沒有分頁/上限提示以外的效能優化；`data/` 三個檔案（共約 4MB）會一次整包載入，還沒做成 `webfonts/wfg-fsung` 那種按需分片。

這些都是後續可以再加的功能，不是遺漏或 bug；`core.js` 的比對邏輯本身已經是完整、經測試驗證的核心。
