# web/ — 部件檢索新實作

取代單一 `.htm` 檔案的新版前端。不用任何 bundler/建置工具——純 ES module + `fetch()`，部署到任何靜態網頁空間後，開啟 `web/index.html` 就能用。

```
web/
├── index.html   頁面骨架（查詢輸入、右側部件鍵盤/字詳情面板）
├── style.css    版面樣式（含深色模式、字源色彩）
├── core.js      比對演算法，逐句對照 legacy/部件檢索.htm 的核心程式碼翻譯而成
├── data.js      用 fetch() 載入 ./data/dt.jsonl、rt.jsonl、vt.json、kt.json
├── data/        dt/rt/vt/kt 資料表本體(含 extract.py 與 schema 說明，見 data/README.md)
├── webfonts/    全宋體(WFG FSung) webfont 切片的 vendored 副本，供 style.css 用相對路徑載入
├── _headers     Cloudflare Pages/Netlify 的快取規則(字型切片長快取，wfg-fsung.css 短快取)，做法比照 ../jigmo-webfonts
├── keypad.js    解析 kt.json 成 { icon, name, rows } 分類陣列，供部件鍵盤使用
├── blocks.js    字源(Unicode 區塊)分類的共用定義：色彩 class、圖例文字、篩選旗標字元
└── app.js       DOM 組裝：查詢輸入、即時搜尋、部件鍵盤、字詳情(含拆分樹渲染)、字源圖例、結果渲染、複製到剪貼簿
```

`web/` 本身沒有任何外部執行期相依（不用 CDN、不用第三方服務）——`data/` 和 `webfonts/` 都是 vendored 進來的副本，整個 `web/` 目錄複製到任何靜態空間都能直接跑。`webfonts/wfg-fsung/` 跟 repo 根目錄的 `webfonts/wfg-fsung/`（那份是給 `wfg-fsung-webfonts` npm 套件用的）內容一致，由 `webfonts/build-wfg-fsung.py` 的 `sync_vendor_copy()` 自動同步；曾經改成指向 unpkg CDN 省掉這份 163MB 的重複，但 CDN 代理 npm registry 的延遲不穩定，改回本機同源字型。

## 為什麼不能用 `file://` 雙擊開啟

跟 `legacy/部件檢索.htm` 不同，這個新版**需要透過 http(s) 伺服器開啟**，不能直接雙擊本機檔案。原因是 `data.js` 用 `fetch()` 讀取 `./data/*`，多數瀏覽器在 `file://` 底下會擋掉這類同源請求。這是刻意的取捨：新實作的目標是「整體部署後，點開能用」，不是延續舊版「可離線雙擊」的約束。本機開發時起一個靜態伺服器即可：

```sh
python3 -m http.server 8000
# 開啟 http://localhost:8000/web/index.html
```

## 功能

- **單一查詢輸入，拆字統一在字詳情面板**。輸入部件序列找字；`\字` 語法直接打開該字的「字詳情」(右側面板，含拆分樹)。曾經做過「部件查字/拆字」兩個分頁，後來發現拆字分頁跟字詳情面板內容重複，砍掉分頁、統一由字詳情呈現。
- **部件鍵盤(頁面級側欄)**：22 個分類(標點筆畫、字頭、包圍、部首類聚等，資料來自 `data/kt.json`，逐字沿用原版 `kt` 分類定義)**一次全部列出**，`position: fixed` 貼住視口右緣、佔滿整個視窗高度、內部捲動，主內容用右側 padding(`--sidebar-w`)讓出寬度。側欄內部**不分欄**：所有分類單欄從上到下排，每個分類是一條 flex-wrap 的「文字流」——行內小標籤 + 該分類全部按鍵攤平連排(原始 `kt` 的分列只是折行提示，這裡直接攤平以求密度)，按鍵縮小到剛好可讀，讓一屏能看到的部件數量最大化。點擊字元會插入到輸入框目前游標位置並觸發即時查詢；面板可收合(展開時只有「收起」鈕會收合，收起後整條可點擊展開)。窄螢幕(手機)改成固定在螢幕底部的抽屜：預設收起成一條、點開彈出半屏高鍵盤，按鍵加大到約 44px 觸控目標；觸屏裝置停用 hover tooltip，碼位資訊併入複製時的 toast。
- **即時查詢(自動完成)**：預設開啟，輸入時debounce 120ms 後自動查詢(IME 組字中不會誤觸發)；即時查詢用較小的結果上限(500，精確命中不受限)，避免大量結果時卡頓；按「查詢」鍵或 Enter 一律跑完整上限(3000)。
- **字源顏色**：查詢結果的字塊用低飽和的字源(Unicode 區塊)淺色底 + 同色系虛線框表達分類；「基本」區佔絕大多數結果，維持中性底色，避免整片結果變成彩虹。圖例直接展開顯示在結果下方，**純顏色說明、不可點擊**——所有區塊一律同時查詢，不提供分區篩選 UI。命中類型(精確/模糊/正體)只用文字顏色/字重表達，不與字源底色互相干擾。hover 字塊會**立即**顯示 `U+` 碼位與分類(自製 tooltip 浮層，不用原生 `title`——原生的有約一秒的內建延遲且無法調整)。
- **「相容」區一律不列出**：「相容表意文字」都是外觀與基本區重複的相容碼位，一般使用者不該把它們打進文件，所以 `app.js` 在使用者沒有自己指定篩選符號時，自動在查詢前面補上「除了相容以外」的區塊旗標(`DEFAULT_FLAGS`)。`core.js` 的旗標語法(`@`/`A`~`J`/`X`/`Y`/`Z`)仍然有效，真的要查相容區可以手動在輸入前打 `X`，但 UI 上不再提供入口。
- **補充字(PUA)提示**：「補充」分類的字使用作者暫定的私有造字區碼位，不是正式 Unicode 編碼，hover 這類字塊只會顯示「補充（尚未正式編碼，暫用私有碼位）」，不會顯示假的 `U+` 碼位；其餘已正式編碼的字，hover 會顯示 `U+XXXX · 分類名稱`。
- **包容異體**、**同字異拆(顯示全部拆法)**、**排除未編碼補充字**：三個核取方塊，語意對應 `core.js` 的 `v`/`d`/`u` 參數。「包容異體」**預設勾選**——拆分表大量使用偏旁變形(𠆢/亻/⺡…)，不開的話「石人工」查不到「砼」(拆分是「石、𠆢、工」)這類情況對一般使用者來說就是「查不到」，關掉屬於進階用法。
- **點字行為選擇 + 字詳情面板**：每個結果字塊(含拆字分頁的節點)是「本體按鈕 + hover 才浮現的小工具列」——直接點字塊＝預設動作(複製到剪貼簿)；hover 出現的浮動工具列可以另外選「查字統網」(開新分頁，沿用 legacy 版預設的 `ref = "https://zi.tools/zi/$ENC$"`)。任一動作都會同時把右側面板切到「字詳情」頁籤，**一次列出**該字的全部資訊，不需要再點按鈕展開：大字、`U+` 碼位/字源、**外部字典大按鈕放頂部**(字統網、漢典、教育部異體字字典、國學大師、萌典、維基詞典中/英/日、Unihan——Unihan 是 Unicode 官方資料庫，再罕見的正式編碼字都有記錄；國學大師的 URL 格式出自其官方說明頁，站點有地區連線限制未能直接實測)、拆分樹。**PUA 補充字沒有正式碼位，外部字典查不到，不顯示字典連結**(面板內以一行說明代替連結，講明原因)，hover 工具列上也沒有字典按鈕。面板若處於收起狀態會自動展開；詳情右上角有 ✕ 可關閉，關掉後回到部件鍵盤；輸入框重新獲得焦點(要繼續組部件)時也會自動切回部件鍵盤。**拆分畫成真正的自上而下節點樹**(org-chart 風格，巢狀 ul/li + CSS 連接線)：`exhaust()` 輸出的括號嵌套文字(`咅(立(亣一┇…)口)⻏`)在 UI 層解析成節點樹，根節點是字本身；內層「┇」的一字多拆分支插入虛線圓的「或」節點分隔。拆分樹尊重「同字異拆」選項(對應黃金案例 `\主` 的行為)：未勾選只畫第一種拆法並提示還有幾種，勾選後全部拆法**左右並排**，容器可橫向捲動。觸控裝置沒有 hover 摸不到浮動工具列，但點字即可看到字詳情，功能不受影響。
- 點擊結果字塊會複製到剪貼簿並顯示提示(toast 帶 `U+` 碼位與字源分類)。
- **字型策略(系統字型優先)**：漢字的字型棧是「系統宋體/明體(Songti TC/SimSun/PMingLiU/Noto Serif…) → WFG FSung webfont → serif」——常用碼位直接用系統字型**零下載**，系統字型缺字的罕用擴充區字由瀏覽器逐字回退到 FSung 的 256 碼位 WOFF2 切片(用到才下載)。**補充字(PUA)例外**：已知是 PUA 碼位的元素(結果字塊/拆分樹/鍵盤按鍵)都掛 `.pua-char` 強制 FSung 優先，因為個別系統字型的 BMP PUA 碼位可能撞車顯示成不相干的字(AGENTS.md 約束 5；資料中 PUA 絕大多數在 15/16 輔助平面，僅 1 字在 BMP PUA)。注意**不能**用 `@font-face src: local()` 做範圍控制——Chrome 86+ 基於反指紋考量不查詢本地字型，實測 headless/真實 Chrome、WebKit 都不匹配(僅 Firefox 部分可用)。實測：只查基本區(`@日月`)零切片下載；含補充字的查詢會正確下載對應 PUA 切片。
- **可分享連結**：查詢內容與狀態同步到 URL query string——`q`(輸入)、`v=0`/`d=1`/`u=1`(偏離預設值的選項才寫入)；舊版含 `mode=tree` 的連結仍相容(自動打開該字的字詳情)。開啟帶參數的連結會還原全部狀態並直接執行查詢。歷史棧策略：「按查詢/Enter 確認過」與「由連結/前後退還原」的狀態視為定案(settled)紀錄，之後第一次變更先 `pushState` 保護它，連續變更(即時查詢逐鍵)只 `replaceState`——打一個詞不會塞進十筆歷史，但「後退」永遠回到上一個定案查詢。

## 已驗證的行為（見開發時的 Node + Playwright 測試）

- `getMatch()` 對「日月／明／日日月／日明」等查詢的結果，與 `legacy/部件檢索.htm` 的演算法邏輯逐句核對一致，且維持「日明 ⊆ 日日月」等原文描述的不變量。
- 「包容異體」勾選後，輸入「咅邑」能查到「部」（`⻏→邑` 的單向映射生效）；未勾選則查不到，只多出「邑」本身。
- 「同字異拆」勾選後，`\主` 能展開出「丶王」「亠土」等全部同字異拆分支；不勾選只顯示第一種拆法。
- 部件鍵盤 22 個分類、699 顆按鍵一次全部渲染(無分頁)；點擊鍵盤字元會正確插入到輸入框並觸發即時查詢。
- 即時查詢會在輸入時自動觸發(debounce)，不需要手動按查詢鍵。
- 補充字(PUA)分類的字塊 hover title 正確顯示「尚未正式編碼」提示，而非誤導性的 `U+` 碼位。
- 預設查詢「豈」會找到 U+8C48(基本字區的「豈」)等 101 字，但不含「相容表意文字」區的 U+F900(外觀相同、但是不同碼位的相容重複字)——證實相容區預設排除；手動在輸入前打旗標 `X` 仍可單獨查到該相容碼位。
- `\字` 解構語法、點字複製到剪貼簿都經瀏覽器（Playwright + Chromium）實測可用，主頁字型（WFG FSung）正確載入渲染，結果依字源正確上色，沒有豆腐字或 console 錯誤。

## 跟 legacy 版比，目前刻意先不做的功能

- 沒有「異體檢索」入口（`core.js` 已經有 `getVariant()`，邏輯已就緒，只是 `app.js` 還沒接 UI）。
- 沒有 cookie/`localStorage` 記住使用者的選項狀態(每次重新整理都會回到預設值)。
- 原版的「跳轉外部字典」已接上(固定用 zi.tools，不像原版 `ref` 可客製化連結)，見上面「點字行為選擇＋字詳情面板」。
- `data/` 幾個檔案（共約 4MB）會一次整包載入，還沒做成 `webfonts/wfg-fsung` 那種按需分片。

這些都是後續可以再加的功能，不是遺漏或 bug；`core.js` 的比對邏輯本身已經是完整、經測試驗證的核心。
